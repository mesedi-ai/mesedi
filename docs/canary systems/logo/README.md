# Canary Systems, LLC — Logo Assets

Drop the logo files here when you save them locally. Suggested filename convention (matches the Verdifax pattern for consistency):

- `canary-logo-original.png` — full-color, original AI-generated render (the goldenrod canary + "CANARY SYSTEMS" wordmark on white)
- `canary-logo-transparent.png` — same, background removed for placement on any color
- `canary-logo-with-bg.png` — version with explicit background for cases where transparency causes issues
- `canary-logo-mark-only.png` — just the canary glyph, no wordmark (for favicons, social avatars, app icons)
- `canary-logo.svg` — vector version (highest priority to obtain; scales cleanly for all uses)

## Design notes (for future reference / brand guideline)

- **Primary mark color:** deep goldenrod / mustard (not bright canary yellow — intentionally more mature/infrastructure-coded)
- **Style:** geometric / faceted / origami treatment, signaling "engineered" rather than "mascot"
- **Wordmark:** clean modern sans-serif, "CANARY" larger than "SYSTEMS" for hierarchy
- **Background pairing:** designed for white background primary; works on dark backgrounds with transparency

## Where the logo will appear

- **Holding-company website** (if/when canarysystems.com or canary.systems is built — optional, holding companies often don't need public sites)
- **USPTO trademark filing** (Class 36 for holding/investment services, optionally Class 42 for tech services)
- **Operating agreement and legal documents** for each subsidiary (e.g., "subsidiary of Canary Systems, LLC")
- **Subsidiary product surfaces** as a co-brand mark ("a Canary Systems company") — small footer placement on Mesedi's marketing site and similar
- **Investor / acquirer materials** when subsidiaries enter sale processes

## Thematic alignment

The canary-in-a-coal-mine metaphor (early-warning system) thematically anchors every subsidiary in the portfolio:

- **Mesedi** = early-warning system for AI agent failures
- **Future products** = ideally continue the "reliability / detection / observability" thread

This shared thematic spine makes the holding-company brand reinforce rather than dilute each subsidiary's individual brand.
